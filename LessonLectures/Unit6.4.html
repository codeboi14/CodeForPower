Unit 6.4: Selection Sort

Selection Sort:
The selection sort is a search and swap algorithm. It is pretty easy to understand, and is an extension and upgrade from bubble sort. Here is how it works:

*Our selection sort process details sorting from least to greatest. For greatest to least, sort vise versa.

 1) The selection sort first spans the array and finds the lowest value.
  Once it finds that lowest value, it will swap its position in the array with the data at the smallest index.
 2) Now, the data at index 0 is sorted in the first search and swap of our selection sort.
   The next smallest index will then swap with the next smallest element in the remaining array etc.

Note that the selection sort will SWAP for n - 1 iterations, where n is the size of the array. (See how we will take 3 swaps to sort the array below, as the array has a length of 4). This applies for all arrays, SORTED or UNSORTED, that undergo selection sort. Lastly, selection sort ALWAYS runs in O(N) squared time, as for each swap, the sort needs to span the entire array to find the lowest value.
