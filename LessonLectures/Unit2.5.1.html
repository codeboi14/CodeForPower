Unit 2.5.1: Inheritance

What is inheritance?
Inheritance is a very powerful tool in OOP (Object Oriented Programming), and is extensively used within Java programs. The idea behind inheritance is a subclass inheriting the methods and fields of its parent class (superclass).

So...why is inheritance so important?
Since inheritance allows classes to obtain the methods and attributes of another class, development time can be drastically reduced. Furthermore, your code will be organized better. Here is an example why:

A superclass is more general, like a vechicle.
A superclass can have subclasses, like SUV, sedan, van, etc.

Inheritance applies here as SUVs, sedans, and vans all have qualities of a vechicle, such as having tires, a engine, etc. Thus, we don't have to reprogram those traits that all vechicles have to those more specific subclasses. We can just make minor adjustments (polymorphism) to fit more specific needs.

How does a class inherit another class?
To inherit, use the "extends" keyword.

Example:

We will create a subclass "Dog" that inherits the superclass "Animal". 
	class Dog extends Animal 
	{
		//code
	}
NOTE: a subclass can only inherit one superclass through the "extends" keyword. Furthermore, inheritance can be blocked depending on the superclass's access modifier (more on this later).

"Is-a" relationship
Superclasses are general templates for subclasses. Therefore, a subclass "IS-A" superclass, and this is ALWAYS true.
In context of our example above, a dog "IS-AN" animal. However, an animal does not necessarily have to be a dog.
We can use this relationship to deternmine whether inheritance between two classes work. For example, we have a superclass, YummyFoods, and a subclass, Cookies. Is the following declaration valid?

 Cookies myCookies = new YummyFoods();

Let's see if this satisfies the "IS-A" relationship! Look from RIGHT TO LEFT. Well, YummyFoods aren't ALWAYS Cookies. Thus, this delcaration is not valid.

Which access modifier allows inheritance?
Only public, protected, and the default access modifier premit inheritance.
Classes with the access modifier of private or final can never be inherited by a subclass.

Another Inheritance Example:
We have a cookie class telling a secret process on how to make cookies! However, we want different "toppings" on our cookie, like chocolate chip, chunky white chocolate, etc.

Thus, we can create a superclass called Cookie that tells how to make different cookies. We can then create subclasses that inherit the Cookie class to make small additional adjustments, like putting on chocolate chips for our luscious chocolate chip cookies.



The "super" keyword
The first statement of a subclass's constructor must include the statement: super().
The purpose of super is to invoke a constructor in the parent class to initalize data.
To call a specific constructor of the superclass with super, put the arguments of the parent constructor into super.
Lets say you have a parent class constructor with the signature: public Point(int x, int y).
To call that constructor with super in the subclass, do: super(x, y).

NOTE: if you don't write super(), Java does it for you by calling the default constructor in the superclass.

Casting with inheritance
You can cast between subclasses objects and superclasses objects in inheritance. When you cast, you are just labeling the object differently. Note that all classes are a subclass of the Object class, and that casting in inheritance is different than casting with primitives. We will use the relationship of the classes in the image below in our casting examples.


Upcasting
You can upcast a subclass object to be treated like a superclass object. For example, when you upcast Ferrari to Car, the Ferrari object doesn't stop being a Ferrari, but it WILL NOT have the properties of a Ferrari. It will just be a plain old car.

 Code:
 Car myCar = (Car)new Ferrari();

NOTE: you can only upcast if the relationship ALWAYS holds up. For example, you can upcast a Ferrari to a car, since a Ferrari is always a car. However, you can't cast a Ferrari to a Tesla, as a Ferrari isn't a Tesla. Furthermore, upcasting is done atomatically between subclasses and superclasses, as Java knows that a subclass can always be a superclass.

Downcasting
Unlike upcasting, downcasting must be done manually since it might fail. A superclass might not always be a specific subclass. For example, you can downcast a car to a Tesla. But, Java doesn't do this for you since a car isn't always a Tesla.

 Code:
 Tesla myCar = (Tesla) new Car();

Summary on Inheritance:
 1) When a subclass inherits the superclass, it also inherits all of the attributes and methods.
 2) A subclass(child) CAN inherit all of the methods & attributes of the superclass(parent).
 3) A superclass(parent) CANNOT inherit the methods & attributes of its subclass(child).

If you think about it, this makes sense. A parent passes his family wealthy onto his children.
However, his children won't pass their wealth onto their parent.
