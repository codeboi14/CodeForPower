Unit 7.1: Method Recursion

Method Recursion
Method recursion is a method calling itself over and over again like a loop. This technique helps break complicated problems into simple problems. Just like loops, the hard part to understanding method recursion is the logic behind it. How and why do we have to call a method over and over again in this specific way?

Example: Let's say you want to create a method that multiplies a range of numbers like a factorial.



Example Explained:
When the factorial() is called, it adds parameter n to the multiple of all numbers smaller than n, and returns the result. When n becomes 0, you reach the end of the factorial, and multiply by one (so the rest of the answer isn't affected).

Tracing the program, we see the following:

factorial(3) will produce:

3 * factorial(3-1)
3 * (2 * factorial(2-1))
3 * (2 * (1 * factorial(1-1))
3 * (2 * (1 * factorial(0))) Note that when n = 0, it doesn't satisfy the if condition, and goes to the else, which makes the method factorial return 1. Any number multiplied by 1 is itself, so our answer isn't affected.

3 * (2 * (1 * 1))) = 6


You see how clever we used method recursion to solve a factorial? Instead of writing a long for loop like to solve a factorial like:

  

We solved a factorial by simply writing:
n * factorial(n-1);


Method recursions are kind of like a computer chip. Looking at the computer chip at a elementary level, it's quite dumb, as it is only capable of understanding 1s and 0s. However, by repeating those 1s and 0s numerous times, you have an extremely powerful chip able to do calculations and run graphic intensive video games. Method recursion is like the same thing. See how elementary this statement n * factorial(n-1) is? However, when you compound these simple things, it becomes very powerful.

Halting Condition
Just like how loops can run into the problem of infinite looping, method recursion can run into the problem of infinite recursion. To stop this from happening, we need a halting condition that tells the method when to stop calling itself.

In our previous example, we stopped the method when the n is not greater than 0, (in other words when n = 0, the method stops performing recursion). Some of the most common halting conditions is when the end is not greater than the start.

Example:
Let's say you want to add consecutive numbers from a start to an end inclusively. Our method name will be sum, so performing sum(1,3) = 1 + 2 + 3 = 6.



Tracing the recursion (so you understand it better):

Here is how sum(1, 3) will work:

3 + sum(1, 3 - 1)
3 + (2 + sum(1, 2 - 1))See how sum(1, 1) won't satisfy the if statement,
which triggers the else statement to return "end", which is 1.
3 + (2 + (1)) = 6
