Other Math Methods:
 1) Math.PI
  This is equivalent to a precise value of pi. You can use it in your mathematical operations.

 2) Math.random()  THIS IS A VERY IMPORTANT MATH METHOD ON THE AP EXAM!

  - The most basic Math.random() method gives your a random double in the range [0, 1).

  - You can type cast Math.random(). For example, type cast to ints and get rid of decimals.

  - To change the range of the random values you want, multiply Math.random() by the numbers of values you want.

  - You can add or subtract ints/doubles to Math.random() to change its lower bound.
  This helps you deternmine what range you want your random values to be.

 Example: return a random dice roll in the range [1, 6] 

   (int)Math.random() * 6 + 1

 Breakdown:
  1) (int) type casts Math.random() to only give integers.
  2) *6 essentially times 6 to [0, 1), making the range [0, 6).
  3) Since the dice roll requires a range from 1 to 6 inclusive,
   you must change the lower bound of Math.random()'s default range.
   +1 to Math.random() * 6 → [1, 7). Since we only have ints, [1, 7) gives us [1, 6].

 General Formula: Math.random()* (number of values) + lower bound



Fundamental Methods of The String Class:
Some methods that the String class provides are extremely fundamental to programming in Java.

We will use this string: String sample = "sample"; in our method examples on the String class below.

Visual representation on what String sample looks like:


 1) .length()

  The length() method returns an int telling how many characters there are in your string.
  *This is basically how long the index of your String is. 
  System.out.print(sample.length());
  Output: 6

 2) .substring()

  The substring method will output a string from your starting index (inclusive) to your ending index (exclusive). 
  System.out.print(sample.substring(0,4));
  Output: "sam"

  NOTE: If you pass an number that is out of your string's index, for example sample.substring(0,7),
  an IndexOutOfBoundsException error will occur.

 3) .indexOf()

  This method takes a substring as a parameter, and returns the first index on where that substring occurs. 
  System.out.print(sample.indexOf("sam"));
  Output: 0

  NOTE: If you pass a substring that doesn't occur, the indexOf method will return -1.

 3) .compareTo()
   compareTo() returns an int indicating which string comes first.

  Given the code:

   

  1) If it returns a positive, then s1 comes AFTER s2

  2) If it returns a negative, then s1 comes BEFORE s2 (in alphabetical order)

  3) If it returns 0, then s1 and s2 are identical (they have no precedence over each other)

How compareTo works:
Starting from index 0, compareTo() compares by looking at each character of the two strings.
It deternmines which character comes first according to their ASCII values.

For example:

 String s1 = "Aaaa";
 String s2 = "bBbb";

 s1.compareTo(s2) will result in a negative, which indicates that the strings are in alphabetical order.
 The little compareTo method looks at 'A' and 'b', and sees that A comes before b according to ASCII,
 and thus returns an answer right away.

 NOTE: compareTo() will keep comparing between characters until the string ends.

Common Things To Know About CompareTo:

 1) "37".compareTo("8") will actually give an answer, as the integers are inside "", meaning that they are strings.
   If it is 37.compareTo(8), it won't work (notice the lack of "").

 2) "Java".compareTo("JavaA") will return a negative, as "Java" is shorter than "JavaA".
   It will return positive if this is the opposite. Note that this will only happen if the
   strings ARE EQUAL up to the end.

 3) ASCII can be tricky!

   "aAaaa".compareTo("Bbbb");

   This will actually return a positive, not a negative! Why? "a" comes AFTER "B" in ASCII values.
   Thus, be very careful when using trickly lill' compareTo()!

  4) Final thing about compareTo()!

   When you use compareTo() to compare strings, it actually gives a PRECISE number in the positive or negative
   range. However, you won't have much use for it, as you mostly care if the number
   is positive (not in alphabetical order) or negative (alphabetical order).
