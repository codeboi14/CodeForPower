Binary Search
Binary search is divide and conquer styled, and much more efficient than a linear search. There is a catch however! The array must be sorted in order for a binary search to work. For example, an array in least to greatest like 1, 2, 3, etc. will work for binary search. An array with values in a random order like 2, 9, 1, -3 etc. will confuse poor little binary search.

Here is the logic behind a binary search:
 1) Divide the array in half, and find the middle value of that array.
 2) Look at the target, is the target larger or smaller than that middle value?
  If larger, discard the lower half of the array. If smaller, discard the upper half of the array.
  *You should see why this will require binary search to have already sorted arrays,
   as this process won't work if the array isn't sorted.
 3) With your array length cut in two, repeat process 1 & 2. If the middle value = target, the search stops.
