Method Recursion WITHIN a Method Recursion
Just like nested loops, you can have methods calling itself within itself. You MUST have a pen and paper to solve these kinds of questions, as it is extremely hard to do this just off memory. Here is a helpful tactic to solve these types of recursion questions: draw a tree, since each recursion branches off until it gets terminated. This is also great preliminary insight on what tree algorithms for competitive programming look like. Take a deep breath, and brute force through the process. You have come so far. This is the last topic in the last unit.

What will idk(10) return? Don't peak at the answer! Try to solve it on your own first.


Here is how you can solve it:

idk(10) = 10 + idk(10/2) + idk(10/4) = 10 + idk(5) + idk(2) 

We need to figure out what the branches, idk(5) and idk(2) will yield.

idk(5) = 5 + idk(5/2) + idk(5/4) = 5 + idk(2) + idk(1)

idk(2) = 2 + idk(2/2) + idk(2/4) = 2 + idk(1) + idk(0) 

See how idk(5) and idk(2) boils down to needing to know idk(1) & idk(0)? 
Now, solve those.

idk(1) = 1 + idk(1/2) + idk(1/4) = 1 + idk(0) + idk(0)

idk(0) = 0, as this condition triggers the if statement. 

Now, trace back up the "tree".

idk(0) = 0

idk(1) = 1 + idk(0) + idk(0) = 1

idk(2) = 2 + idk(1) + idk(0) = 2 + 1 + 0 = 3

idk(5) = 5 + idk(2) + idk(1) = 5 + 3 + 1 = 9

idk(10) = 10 + idk(5) + idk(2) = 10 + 9 + 3 = 22

22 is our answer. When doing method recursion, especially for linear ones (not the nested one we showed you above), you should be able to see PATTERNS within recursion statements so that solving it will be as quick as 1, 2, 3.
