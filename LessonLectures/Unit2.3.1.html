Unit 2.3.1: Java Modifers

Modifiers are split into two groups: access & non access.

Access Modifiers
You have probably seen the word "public" for quite a few times when writing classes and methods. Well, the public keyword is actually an access modifier setting the ACCESS level (who has the ability to call and use that block of code) for those classes and methods. Note that access modifers can also apply to constructors, attributes, etc.


Besides the default modifier, going down the chart results in increased "security" for your code.
In the AP Computer Science A course, you will just need to know how to use public and private.

Using Public
You should use public if you want your code to be accesible to other classes.
For example, you created a helpful Math class that does mathamatical operations, like: pow, sqrt, etc.
You probably want to declare that Math class as public so you can use it in your other classes, like a class that solves the quadratic equation.

Using Private
The instance variables(attributes) in your classes should be declared as private.
Why? Let's say you created a helpful class performing the quadratic equation.
If you declare variables in that class as public, other classes oustide can
change your lovely variables and mess the code up!
Furthermore, other programmers have no reason to access your personal variables.
They just need to pass in their equation for your quadratic equation code to solve.


Non-Access Modifiers:
There are two non-access modifiers that you should know: final & static.

 1) final

  The final modifier means that something can never be changed.

  For example, you have a attribute called pi. Well, pi is always 3.14, so you don't want that to be altered.
  Thus, you can declare the attribute like this:

  final int pi = 3.14;
  //no one can change the value of our pi int.

This simply means that that code inside the {} are a part of the method.
