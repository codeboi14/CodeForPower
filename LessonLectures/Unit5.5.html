Unit 5.5: ArrayList Methods

ArrayList Methods
Unlike arrays, the ArrayList provides methods to access, manipulate, remove, and add data.
These methods are fundamental to using ArrayLists, and you are required to memorize them for the AP exam. This might seem a bit daunting, but you will quickly see how EASY and straight-forward these methods are.

Adding Elements:

ArrayList Method: add(Obj)

Instead of justing adding an object to the end of the ArrayList, this method adds an object to a specified index.

Removing A Specific Object

ArrayList Method: remove(obj)

This method searches through the ArrayList for the object you passed. If it occurs, the first occurrence of that object is removed, and true is returned. If the object you passed doesn't exist, false is returned, and nothing is removed.

Note* The remove methods are sometimes referred to as void methods, as on numerous occasions, you don't care what they return. You just need the element removed from the ArrayList.

LOGIC ERRORS:

You have to be extremely careful when using the remove() method, as LOGIC ERRORS might occur. The question below showcases this.

Question: Given an already declared ArrayList called myNum, which contains the Integers: 2, 2, 2, 2, 5, 5.
How do you remove all even digits in the myNum ArrayList?

Our Solution:



This code might seem good on the surface, but it won't give us our desired answer! Here's why:

 1) At first, i (which stands for index) is 0. You successfully remove the even digit 2 from the ArrayList.
  The whole ArrayList now becomes: 2, 2, 2, 5, 5.
 2) i gets incremented to 1. HOWEVER! At index 1, you SKIP the even digit at index 0.
  You remove the even digit at index 1 for the ArrayList. The ArrayList now becomes: 2, 2, 5, 5
 3) i gets incremented to 2. HOWEVER! The even digit at index 0 and 1 is SKIPPED and not removed.
  At index 2 of the new ArrayList: 2, 2, 5, 5, the digit is odd, so nothing gets removed.
 4) i gets incremented to 3. At index 3, the digit is odd, so nothing gets removed.
  The loop then finishes when i goes to 4, as 4 is not less than the length/size of the current ArrayList.

What gets outputted is an ArrayList containing the digits: 2, 2, 5, 5. This is not what you want! 
How To Overcome This Logic Error:
