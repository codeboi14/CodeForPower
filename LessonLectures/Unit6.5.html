Unit 6.5: Insertion Sort

Insertion Sort:
Insertion sort is a bit more complex than selection sort, but it is more efficient. Here is how it works:

NOTE: our process details insertion sort sorting from least to greatest.

 1) Rather than traversing the entire array, the insertion sort compares two adjacent elements,
  and depending on the comparison, inserts the second value in front of the first value if its unsorted.
 2) The sort checks the third element with the second element. If it's not in order, insert it in the correct position.
  This process continues until the whole array is sorted.

Step 1: compare the value in index 0 with that in index 1 (adjacent elements). Since, 9 > -1, the elements are greatest to least, not our desired least to greatest. Since it's unsorted, we will insert -1 before 9. (We will tell how this insert process works in greater detail later).



Step 2: compare the value in index 1 with that in index 2. Since 9 > 3 (unsorted), we will have to insert 3 in the correct position. Insert by comparing 3 with -1. -1 < 3 (sorted) so put -3 after -1.



Step 3: compare the value in index 2 with that in index 3. Since 9 > 2 (unsorted), we will have to insert 2 in the correct position. Insert by comparing 2 with previous array elements. 3 > 2 (unsorted), so compare 2 with -1, -1 < 2 (sorted) so place 2 after -1.



Step 4: compare the value in index 3 with that in index 4. Since 9 > 4 (unsorted), we will need to insert 4. Insert by comparing 4 with 3. 3 < 4 (sorted) so place it after 3.



Inserting: Notice how when we are inserting, if the symbol is GREATER THAN, it is unsorted, so you have to keep checking where to insert. If the symbol is LESS THAN, it is sorted, so we just have to insert the value after that element. Furthermore, notice how we have to SHIFT some values to the right to make room for the insert to happen.

Insertion Sort Algorithm (Least to Greatest)
In terms of loops, the insertion sort will be shorter, as it optimizes which values to compare. In its worst case, insertion sort will take up to O(N) squared time. Note that the selection sort requires n-1 passes (as in swap), and the insertion sort also requires n-1 passes (as in insert). See how we took 4 inserting steps to sort the array above that had a length of 5.
