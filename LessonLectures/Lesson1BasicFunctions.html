Strings I. The Basics:
Strings are a series of characters (words) denoted by "". Because strings are so commonly used, you might assume it is a primitive data type, but it is not. Strings are actually OBJECTS (more on objects in Unit 2).

Examples on Using Strings:

 1) You can declare strings

    String myString = "hiii! how are you?";

  2) You can print strings with the statement: System.out.print or println
  NOTE: println adds a new line AFTER your string.

    System.out.println("my string");

    Outputs: my string

  3) You can CONCATENATE strings with "+" (adds two strings together)

    String myString = "finish this-";
    String addThisString = "phrase";
    System.out.print(myString + addThisString);

    Outputs: finish this-phrase

NOTE: If you have a string in a statement, using "+" to add integers will concatenate them instead.


  For example:

   System.out.println("5 + 5 = " + 5 + 5) ;
   Outputs: 5 + 5 = 55
   (concatenates the ints to a string! Noooo!)

To avoid this, use parenthesis (see that 5 + 5 = 10 now).
System.out.println("5 + 5 = " + (5 + 5) ) ;
Outputs: 5 + 5 = 10

String Indexes:

There is an index to every String character.

For example:
String = "myString"; has the following index for each character: 

Index: 0, 1, 2, 3, 4, 5, 6, 7

       m  y  S  t  r  i  n  g
See that each character corresponds with an index. Every index starts at 0.

Why In The World Do We Need To Know Indexes?
The index of a String has many purposes. For example, you want to find a sub-segment of the String, do loops with the String, etc. You will realize the full importance of indexes when you start coding in depth.

Escape Sequences
The backslash (\) is an escape sequence that can perform special things.

Must know escape sequences:

 1) \n is used to indicate a new line

   Example: System.out.print ("escape" + \n + "sequence");

   Outputs: escape
        sequence

 2) \" is used to literally print quote marks.
   \\ is used to literally print \.

   You can't print these special (", \) characters anyway else.

   Example: System.out.print("hi + \\ + \"");

   Outputs: hi + \ + "

  3) \t may be used to indent when printing


Arithmetic Operations:
Java can do math! The symbols +, -, *, and / represents addition, subtraction, multiplication and division, respectively. The % "modulus" symbol gives the numerical remainder of division.

NOTE: when doing division, java "truncates" the answer. For example, 4/3 = 1 to java, as it savagely eliminates the decimal. If you want Java to give a decimal answer, use double instead of int.

Example:
  int a = 10;
  int b = 10;

Doing the following in Java yields:
 a + b = 20 (addition)
 a - b = 0 (subtraction)
 a * b = 100 (multiplication)
 a / b = 1 (division)
 a % b = 0 (modulus)



Precedence:
Java follows precedence just like regular math. Division and multiplication is performed first, unless there are parentheses.

 3 - (4/2) = 1 in Java.

Casting:
Casting is a process where data is forced into another type of data. Put the data you want to cast to in parentheses.

Example: changing an int to a double

  System.out.print((double)2 );
  *prints 2.0 instead of 2

  System.out.print((double)(4/3));
  *prints 1.0 (does 4/3 in int first, then casts)

NOTE: you can cast any primitive data type to another, but use logic when doing so. For example, you can technically cast ints to booleans, but it doesn't make sense (sometimes).

Operators I. Pre and Post Increment

1) Post-increment operator (x++)

  y = x++ is equivalent to performing:
  y = x then x = x + 1

2) Post-decrement operator(x--)

  y = x-- is equivalent to performing:
  y = x then x = x - 1

3) Other operators +=, -=, *=, /= & %=

 a+=b is equivalent to a = a + b
 a*=b is equivalent to a = a * b
 etc.

4) Pre-increment operator (++x)

 y = ++ x is equivalent to performing:
 x = x + 1 then y = x

5) Pre-decrement operator(--x)

 y = --x is equivalent to performing:
 x = x - 1 then y = x

NOTE: the order that x and y get assigned are flipped in pre and post increment/decrement. Because of this, be careful of what you use, as it may result in logic errors.
