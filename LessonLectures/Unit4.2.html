The do-while loop
The do-while loop follows a slightly different execution sequence than the while loop. The do-while loop checks the boolean condition AFTER executing its code block, contrasting the for and while loop, which checks the boolean condition BEFORE. It is because of this that do-while loops are guaranteed to execute at least once. The keywords "do" and "while" are necessary to declare the do-while loop.

Syntax:

  do
  {
     //code block
    //increment/decrement
  }
  while (boolean condition);

Do-while loop execution sequence

  1) Execute code block within "do"
  2) LCV is updated inside "do"
  3) check if the new LCV satisfies the condition statement inside "while"
   - (if it satisfies the boolean condition in the "while" statement (true)) → Go back to #1
   - (if false) → break the loop

Do-while loops are also called "exit control loops", as they exit the loop to check for the control (use see this with the separate code (inside "do") and boolean condition statement (inside while)). If you still have trobule comprehending the do-while loop, think about it like this: do the code block inside the loop while the boolean condition is true.

Lastly, the do-while loop can "technically" be converted into a while and for loop if you want. Why technically? Well, since the code block within "do" is always executed first, the loop is guaranteed to run the code block atleast once, even if the boolean condition is false.
