Unit 2.2: Objects

Java is OBJECT-oriented programming, so you can probably tell how important objects are. An object is an instance of the class, which can contain attributes and methods (functions). It is initialized with the "new" keyword, which invokes a special method called a "constructor".

To create an object, specify the class name, the name you want to give your object, and the keyword new.

 Example:
   Scanner myScanner = new Scanner(System.in);

 Breakdown:
  1) class name: Scanner
  2) object's name: myScanner
  3) = (assignment symbol)
  4) new keyword
  5) class name: Scanner
  6) parameters (whats inside the parenthesis): System.in

(You will learn more about parameters in our methods chapter).

NOTE: You can have as many objects as your heart desires in your program.

Object Reference (Important Concept)
Since a object takes up a lot of memory space, a reference to the object and not the object itself is stored in the memory. This is unlike primitive data types, where the actual value and not its address in the memory is stored.

 For example:
   int num = 10;

  "num" actually contains 10, and not where it is in the memory, like location 10@001020.

Since the memory location for an object is stored instead, logic errors might occur! This reference concept is tested on the AP computer Science A exam!

Null Reference:
Null means not referring to any object (null = nothing). Note that object variables are automatically initialized to null in Java if you don't assign them to something.

ex) Scanner myScanner; (this is a null reference).
