JAVA OOP:
OOP stands for object-oriented programming. This type of programming revolves around creating objects that may contain data or methods (more on objects & methods below).

Benefits of OOP programming:
  1) Easier to debug (finding logic errors in your code)
   Because of their debugging superiority, OOP languages like Java, C++, etc.
   are commonly used in competitive programming and coding software for real-world apps.
  2) Code can be reused, leading to shorter development time and shorter code.
  3) OOP languages have a very clear structure when programming.

Classes & objects are two main aspects of OOP programming. Here is a simplified look on what their concepts are.

Java Classes
Classes are the big picture of java. A class is a group of statements assembled into a single unit.
It is the blueprint of the object, and is stored in a file.

Structure of A Class:

 public class HelloWorld
 {
    //code not shown
 }

Naming Classes: 
	1) All classes should begin with a capital letter. 
	   This is proper naming convention.
	2) You must follow the same rules when naming 
	   variables when you name classes. 
	   Names must start with a letter & not a number, 
	   they should also not contain any special characters, spaces, etc.
NOTE: code is always executed top-down in Java. So line 1 is performed first, then line 2, etc.

Think of Classes Like This:
Assume that we created a math class. What does the math class do? Well, the math class can have methods that contain specific code for doing square root, exponent, etc. The class doesn't really do anything, it just organizes related code, like mathematical operations, so it can be easily used in other other programs & classes.




Terminology:

Driver Class: A driver class is created to control a large program, as it drives the program through its general structure.

Object Class: This is like the "guts" of the methods that the driver class calls. It defines what the object has and does.

Think about driver & object classes this way:
 When driving a car, the "driver" operates the pedal, which makes the car do something, like moving.
 What happens under the hood when you press the pedal is what the object class does.

Abstraction
Abstraction is knowing what something does, but not how it does it. For example, you know that your brain can tell the difference between a bee and a three. However, you don't know how it knows the difference, since you just know that creature is a bee, and you just know that symbol is a three. Your thinking is abstract.
