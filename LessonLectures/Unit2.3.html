Unit 2.3: Methods

What Are Java Methods?
A method is a group of code that performs a SPECIFIC task. They are used to make code shorter and easier to read. Furthermore, methods are always created within a class, and are identified by the Java Compiler by its syntax of having a modifier, return type, name, and parenthesis (which holds parameters if they exist). These will be discussed in greater detail below.

Introduction to Modifiers
Modifiers determine how the method can be used, and who gets to access to them. There are two types of modifiers in Java: access and non access modifiers. In short, access modifiers determine the scope of the method, and non-access modifiers determine the method's inheritance capabilities (more on inheritance later).

Return Types
Return types determine what the method outputs after it performs a certain action. Return types can be anything from data structures (arrays, arrayLists, etc.) to data types (ints, doubles, booleans, Strings, etc.). If a method does not return anything, the keyword "void" can be used to signify that. Methods that don't return anything but change an instance field (attribute) of a class are known as mutator methods, or setters. Methods that return the value stored inside an instance field are known as accessor methods or getters.

Note that you can return anywhere in your code- this isn't just limited to methods. For example, you can return at the end of an if else, for loop, etc. However for the AP exam, return will just be used in methods.

Method Names
Method names help Java (and you) identify different methods. To properly name methods, follow the same naming convention when naming variables.

Note that two or more methods can have the same name! However, their parameters must be different so that a distinction can be made between the two methods. This is called method overloading (we will explore this concept in greater detail later).

How are Java methods declared?
All methods have 5 components to them; all of which are listed below.
 1) Modifier
 2) Return Type
 3) A Name
 4) Parameters
 5) Body

The syntax for declaring a method is shown below. Note that the method name and its parameters are called the method signature, as these are the components that identify a method. When you use a method, the method signature will be called.

 Syntax:
  accessModifier nonAccessModifer returnType name(parameters)
  {
    //your code
  }
