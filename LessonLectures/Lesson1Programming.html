Compiling & Errors
Computers only understand 1s and 0s, so they don't understand your
Java code. Thus, an interpreter is used within the developer environment
that translates your code to a language understood by the machine. This
process is called compiling.

An error occurs when your code can't compile. There are 3 types of errors:

 1) Syntax error: simplest to solve, just follow the correct format.

   Example) To print a statement, you should type:

   System.out.print("statement goes here");

   However, if you write Stem.out.print("hi"); , this won't compile.

  2) Run-time error: slightly harder to solve. An example can be making the program
  loop indefinitely(more on loops in later chapters, don't stress this part).

   for(int i = 10; i > 0; i++){
     //program does stuff
   }

 *notice that "i" can never reach a value less than 0 (condition to end the loop)
 as it is added when it should be subtracted. As a result, this loop continues indefinitely.

 3) Logic error: this will be pretty hard to solve, as the literal logic of
 your code has errors. A simple real world logical error can be trying to call
 your friend with a rock instead of a phone.

Starting off, see that code is always executed top-down in Java. So line 1 is performed first, then line 2, etc.

 1. The class is a fundamental aspect of Java programming, as it is the blueprint of an
  object(this concept is explained in depth on Unit 2). The class in the program we wrote is:

 2. This program uses methods from the Java Library:

    System.out.print("Hello World");

   The Java Library is a collection of pre-written classes & methods that programmers
   may reuse. The library makes performing many things easy, such as,
   printing something. Instead of writing that lengthy code to physically print something, the
   library provides the pre-coded method: System.out.print(), which makes doing
   the operation very simple.

  3. Programmers may create their own classes, which can contain numerous
 programmer-defined methods (code that does a specific thing, for example, taking the square root).

 4. public static void main (String[] args)

   This is a special method called the "main" method.
   This is used to run your program. Just write this word for word- you will
   understand why we must do it in this way later.

Note* The first statement EXECUTED in every Java program is always the first statement in the main method.

  5. All the statements in a class or method definition must be enclosed in a
 pair of curly braces {}, which represents where that code segment starts and ends.

 6. Java is case-sensitive, meaning that the words Public & public are
 completely different things to poor little Java.

  - Java has reserved words (keywords) such as public, class, static, void etc.
  that use lowercase letters only. These words have special meaning to Java and
  may not be used for anything else.

  - Java identifiers are not keywords.

  - Class names from the Java Library (String, System, etc.), and programmer-defined
  classes (like the HelloWorld class we did) begin with an uppercase letter.

 7. All Java statements must end with a semi-colon.
