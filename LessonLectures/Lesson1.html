LESSON 1: PRIMITIVE DATA TYPES:

What Is Data?
Data is information, such as numbers, conditional statements (yes, no), etc. For computers to process such information, data types and data structures exist.
(NOTE* computers don't understand data. They just have access to the ones you store.)

Primitive Data Types:
There are 8 primitive data types that are predetermined and built into Java. These data types are fundamental to coding, and have a predetermined memory size allocated for their storage. Listed below are the 8 primitive data types in Java. The ones circled are the only ones you need to know.
NOTE: floating-point types should never be used to store exact values due to small accuracy errors.


Syntax For Primitive Data Declarations
(primitive type) dataName = (data that the type can store);

 Example: int JavaIsTheBest = 1;

 Recap: primitive types = int, double, boolean, byte, short, long & float

 Note* You can declare a variable, and assign it to an value later.
 Example:
    int x;
    x = 1;

Printing Statements In Java
Prehaps the first thing you will program is printing something in Java. Printing is extremely useful since you can see your code's output. To print, you must write out the statement below word for word:

 System.out.print(your statment goes here);

What gets printed is what's inside the parenthesis.
To print text, put quotes around your statement:

 System.out.print("My statement"); This simply outputs: My statement

Why Do We Print Like This? Just memorize this statement for now. We will explain why later.



int (integer type)
Ints are one of the most frequently used primitive data types. It stores an integer (such as 2,100,467). You cannot store ints over their upper and lower boundaries, as this will cause an overflow error.

NOTE: the int can overflow should you perform arithmetic with integers that store the largest or smallest numbers that the int can store. For example, if you decide to assign an int to a the value 2,147,483,647 + 1, it will overflow, since the new number is larger than the max possible value that the int can store. Furthermore, ints can have underscores between its digits for readability, like 99_012.

double (floating-point type)
Doubles are another frequent data type that Java programmers use to store decimals (3.1415926). Typically, any decimal will be expressed as a double instead of a float, as memory size isn't a problem that concerns modern day computers. Doubles are more powerful than ints as they can store larger numbers, and ints aswell! In doing so, the integer stored will be promoted to a double through the appending of zeros after the units digit like: 32 → 32.0
Doing calcuations with a double will result in an extremely small accuracy error, like: 32.0 + 0.0 = 32.0000000000000001. Therefore, doubles should never be used to store precise values.

NOTE: Just like ints, doubles can overflow should you perform arithmetic that exceeds the double's largest and smallest values.

char (character type)
Characters are another frequently used primitive data type that programmers use in Java. Char data can store all the letters of the alphabet in either lowercase or uppercase (a-z, A-Z), along with ASCII values (more on ASCII later). The syntax for char is a bit special, as it requires single quotes that encase the char data.

Example: char myCharName = 'a';

NOTE: Character data types rely on the ASCII system and therefore, can be incremented and decremented like an int. Conversions between chars and ints are possible, and will be discussed in further detail later.


boolean (true or false)
Booleans can only store true and false values, which can be expressed by 1 (meaning true) and 0, (meaning false). Booleans are mostly used for logical decision making. For example, if "food is yummy" = true, then I will eat.

Naming Convention for Variables:
All of your data types(variables) need a name. Why? Well, Java is asking you why do you have a name? The reason why is so people can call you to send information (like, hey! Let's go get icecream!).

NOTE: names are also called identifiers. Java allows you to use names to call(obtain) data stored in the memory of the computer, instead of a nonsensical memory address. This makes your job a lot easier.

1) Names MUST begin with a letter or the symbols: "_" & "$".
After beggining with a letter, you may include numbers in your name.

  Example) int x123_underscoresAreAllowed = 0;

2) Variable names must not contain special characters like * or spaces.

3) An identifier(variable name) should have a logical name, and start with a lowercase letter (this doesn't affect the code though).

  Example) Don't name a variable:
       int bogusNonsense = 0;

4) Use camelCase with each subsequent word for readability. First letter of the subsequent word begins with a capital letter.

  Example) int camelCaseIsCool = 0;
